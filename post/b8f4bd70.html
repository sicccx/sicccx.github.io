Hexo博客搭建

发表于2022-09-26|更新于2022-09-26
|字数总计:2.5k|阅读时长:10分钟|阅读量:

本文记述了小白从零开始搭建自己的博客之旅，使用Hexo博客框架、Butterfly主题、部署于Github上

一.Hexo与Markdown

1.Hexo与Markdown介绍

Hexo是一个简洁的博客框架，生成静态博客网页，md文章可以很好的被渲染，所以用Hexo搭建博客、markdown来写博客非常适合我这样的小白。

Hexo可以使用很多主题，我使用的是butterfly主题（认识的很多大佬用过，感觉很好看！），配置项也非常多，非常适合二次元

markdown写文可以使用：

  • 各种支持markdown的笔记软件（分屏单屏都有）
  • **MarkdownPad2**：有一种官方感
  • **Markdown网页（分屏）**：挺好用的，但是不能云同步，写文记得自己保存备份！
  • VSCode+Git（推荐）：VSCode使用 Markdown All in One、Markdown Preview Mermaid Support、markdownlint插件来浏览md文件，使用 office Viewer(Markdown Editor)插件来编写md文件，Git来云同步文章文件。感谢写这个插件的Weijan大佬。

2.安装

hexo所需前置：

  • Node.js
  • npm：npm install -g hexo-cli进行cmd安装，也可也直接在Github上下载源码压缩包
  • npm install hexo-deployer-git --save 这个插件是用于Hexo博客部署到Github页面上，插件的安装要在hexo主目录
  • npm install hexo-abbrlink --save 生成永久url插件

hexo目录结构：

  • _config.yml：hexo配置文件
  • themes： hexo主题存放位置
  • node_modules ： npm安装存放位置
  • package.json： 记录npm安装信息
  • package-lock.json： 记录npm安装信息
  • scaffolds： 模板文件夹
  • source： 存放文章
    • post： 存放md文件
    • public： 存放md文件转换成的html文件

在hexo主目录右键打开git bash命令行，依次运行 hexo clean hexo g hexo s，启动后在浏览器进入 localhost:4000本地渲染博客、测试页面。

hexo的配置都在 _config.yml文件里

二.Github

1.Github基本操作

  1. 克隆仓库： git clone 仓库名

  2. 上传代码：

    1. 建立本地git仓库：git init
    2. 将文件添加到仓库：git add . # 添加所有文件
    3. 提交到本地仓库：git commit -m "上传注释说明"
    4. 本地关联到Github：git remote add origin https://github.com/xxxxx/xxxx.git
    5. pull合并master分支：git pull origin master
    6. 如果上一步报错fatal: refusing to merge unrelated histories，说明仓库不同，通常是分支重新命名后冲突，则使用：git pull origin master -allow-unrelated-histories允许合并
    7. 上传本地仓库代码：git push -u origin master

  3. 更新代码：

    1. 查看仓库状态：git status
    2. 更新全部：git add *
    3. 更新说明：git commit -m "注释"
    4. 拉取当前分支最新代码：git pull
    5. 修改完代码后：git push origin master

2.博客挂载

前置条件：npm install hexo-deployer-git --save 已经安装了该插件

如果前面没有问题，访问：xxxxx.github.io，就可以看到自己的博客页面了，不过显示的应该会是Hexo默认页面

有些部件渲染可能没这么快，比如live2d。我第一次更新页面后发现live2d根本没有，明明本地部署看得见，结果过一会儿打开又有了，邪门（

踩了个坑：本地的博客挂载到Github后，Github文件和本地博客源文件不一至。部署后Github里的是展示文件，如果想用Github云同步备份博客源文件，方便其他设备使用，得另外新建一个仓库（或分支？），把博客源文件复制到另外一个文件夹新建仓库备份（因为原来的部署源文件文件夹里已经连接上展示的io仓库了）

三.Butterfly主题

部署完Hexo默认博客页面完全没有达到我对自己博客的预期！

所以美化Hexo博客也很重要：本文通过Butterfly主题示例，其他好看好用的主题也有很多

1.主题安装

  • 在Hexo根目录git：git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
  • 如果要安装比较新的dev分支：git clone -b dev https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
  • 更新主题：在主题目录下 git pull
  • 如果已经下载了butterfly主题文件，放入到themes文件夹即可（landspace是Hexo自带的默认主题）
  • 必备插件pug 以及 stylus 的渲染器：npm install hexo-renderer-pug hexo-renderer-stylus --save

2.应用主题

修改Hexo配置文件：theme: butterfly

3.Butterfly配置项

配置文件在主题文件夹里，butterfly目录里的 _config.yml

建议先将此配置文件复制粘贴一份，重命名为 _config.butterfly.yml，部署时会优先此配置文件与原配置文件合并使用，在butterfly主题更新后_config.yml会被覆盖掉，这样不用麻烦重新修改配置文件。
